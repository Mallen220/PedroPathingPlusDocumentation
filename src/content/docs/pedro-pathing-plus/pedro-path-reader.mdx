The `PedroPathReader` is a utility class in PedroPathingPlus that allows you to dynamically load path data from `.pp` project files generated by the [Pedro Pathing Visualizer](/pedro-pathing-plus-visualizer/getting-started/).

Instead of hardcoding complex geometry and poses into your Java classes, you can design your paths in the visualizer, save the `.pp` file to your robot controller, and load the points at runtime.

## Overview

When you export a **Command-Based (SolversLib/NextFTC)** OpMode from the visualizer, the generated code uses `PedroPathReader` by default. This approach separates your path logic from your execution code, making it easier to adjust paths without having to recompile and deploy your app every time.

## Prerequisites

To use `PedroPathReader`, you must:
1. Export your project from the visualizer as a `.pp` file (using the Export dialog or `Ctrl+Shift+X`).
2. Upload the `.pp` file to your robot's storage (typically the Control Hub or phone). The easiest way to do this is using the FTC OnBotJava interface or Android Studio's Device File Explorer. The file should be saved to the standard app directory or assets.

## Basic Usage

### 1. Initialization

To initialize the `PedroPathReader`, you need to provide the name of the file and the Android `Context` (which can be obtained from the `HardwareMap`).

```java
import com.pedropathingplus.PedroPathReader;
import com.pedropathing.geometry.Pose;
import com.qualcomm.robotcore.hardware.HardwareMap;

// ...

public void initPaths(HardwareMap hw) {
    try {
        // Initialize the reader with the filename and application context
        PedroPathReader pp = new PedroPathReader("AutoPath.pp", hw.appContext);

        // ...
    } catch (IOException e) {
        e.printStackTrace();
    }
}
```

### 2. Loading Poses

You can retrieve named poses from the `.pp` file using the `get()` method. The names must match the names of the points/paths you assigned in the visualizer.

```java
// Load poses by their name in the visualizer
Pose startPoint = pp.get("startPoint");
Pose scorePreload = pp.get("scorePreload");
Pose parkLocation = pp.get("parkLocation");

// Set the starting pose for the follower
follower.setStartingPose(startPoint);
```

### 3. Building Paths

Once the poses are loaded, you can construct your `PathChain`s as usual using the Pedro Pathing `PathBuilder`.

```java
public void buildPaths() {
    PathChain startToScore = follower
      .pathBuilder()
      .addPath(new BezierLine(startPoint, scorePreload))
      .setLinearHeadingInterpolation(
        startPoint.getHeading(),
        scorePreload.getHeading()
      )
      .build();
}
```

## Generated Code Example

When you use the Command-Based export in the visualizer, it generates a complete structure that utilizes `PedroPathReader`. Here is a simplified version of what that looks like:

```java
public class AutoPath extends SequentialCommandGroup {

    private final Follower follower;

    // Poses
    private Pose startPoint;
    private Pose point1;

    // Path chains
    private PathChain startPointTOpoint1;

    public AutoPath(final Drivetrain drive, HardwareMap hw) throws IOException {
        this.follower = drive.getFollower();

        // 1. Initialize Reader
        PedroPathReader pp = new PedroPathReader("AutoPath.pp", hw.appContext);

        // 2. Load poses
        startPoint = pp.get("startPoint");
        point1 = pp.get("point1");

        follower.setStartingPose(startPoint);

        // 3. Build the actual paths
        buildPaths();

        // 4. Add commands to the sequence
        addCommands(
            new FollowPathCommand(follower, startPointTOpoint1)
        );
    }

    public void buildPaths() {
        startPointTOpoint1 = follower
            .pathBuilder()
            .addPath(new BezierLine(startPoint, point1))
            .setLinearHeadingInterpolation(
                startPoint.getHeading(),
                point1.getHeading()
            )
            .build();
    }
}
```

## Troubleshooting

- **File Not Found Exception**: Ensure the `.pp` file has been uploaded to the correct location on the robot and the filename is spelled exactly as it appears in the code (it is case-sensitive).
- **Null Pointer Exception when getting a pose**: This means the `PedroPathReader` could not find a pose with the exact name you requested. Double-check the names of your paths and points in the visualizer. By default, the visualizer assigns names like `startPoint`, `point1`, `point2`, etc.
