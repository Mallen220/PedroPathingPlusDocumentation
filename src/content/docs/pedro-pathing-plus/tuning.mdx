# Tuning Pedro Pathing

Tuning is the process of adjusting the `PIDFCoefficients` (Proportional, Integral, Derivative, and Feedforward) so that your robot's physical movements match the mathematical paths generated by the library. Proper tuning is crucial for accurate and reliable autonomous routines.

Pedro Pathing uses three main sets of PIDF controllers for its path follower:
- **Translational PIDF:** Controls the robot's movement along the X and Y axes (driving forward, backward, left, right).
- **Heading PIDF:** Controls the robot's rotation/orientation.
- **Centripetal Force PIDF:** (Optional but recommended) Compensates for the outward force when the robot turns while driving.

These are configured in your `Constants.java` file (or equivalent) using the `PIDFCoefficients` and `FilteredPIDFCoefficients` classes from `com.pedropathing.control`.

## The Tuning Process

A recommended approach for tuning a holonomic drivetrain (like Mecanum) is to tune in this order:

1.  **Translational PIDF**
2.  **Heading PIDF**
3.  **Centripetal Force PIDF** (If needed)

### 1. Translational Tuning

The translational controller ensures the robot reaches the target X, Y coordinates accurately.

- **Target:** Have the robot drive a simple straight line (e.g., 24 inches forward).
- **P (Proportional):** Start by increasing `P` until the robot reaches the target distance reasonably quickly but without violent oscillations. If it stops short, `P` might be too low. If it overshoots heavily and jitters, `P` is too high.
- **D (Derivative):** Once `P` is close, increase `D` to dampen any remaining oscillations and smooth out the stop. `D` acts as a brake as the error decreases.
- **I (Integral):** Generally, keep `I` at `0` for FTC drivetrains unless there is a persistent, small steady-state error (e.g., it consistently stops 0.5 inches short). If used, keep it very small to avoid integral windup.

### 2. Heading Tuning

The heading controller ensures the robot maintains or reaches the correct angle.

- **Target:** Have the robot rotate in place (e.g., 90 degrees or 180 degrees).
- **P (Proportional):** Similar to translation, increase `P` until the robot snaps to the target angle quickly. If it spins past the target and oscillates, `P` is too high.
- **D (Derivative):** Increase `D` to dampen the rotation as it approaches the target, preventing overshoot.
- **I (Integral):** Usually kept at `0` for heading as well.

### 3. Centripetal Force Compensation

When driving in a curve, physical robots tend to drift outward from the path. This controller applies a corrective force towards the center of the curve.

- **Target:** Have the robot drive a circular or strongly curved path.
- **Tuning:** This is often a single proportional coefficient. Increase it until the robot tracks the curve tightly without drifting outward. If set too high, the robot will cut the corner too sharply.

## Understanding `PIDFCoefficients`

The `PIDFCoefficients` class allows you to set the standard terms:

```java
// Example: P = 0.1, I = 0, D = 0.01, F = 0
PIDFCoefficients myCoeffs = new PIDFCoefficients(0.1, 0, 0.01, 0);
```

### Advanced: `FilteredPIDFCoefficients`

For more advanced control, Pedro Pathing includes `FilteredPIDFCoefficients`. This applies a low-pass filter to the derivative term or other signals, which can help smooth out noisy sensor data (like velocity readings from odometry) and allow you to use a higher `D` term without introducing high-frequency jitter.

## Best Practices

- **Use Telemetry:** Log the robot's current Pose (`X`, `Y`, `Heading`) and the Target Pose to your Driver Station or dashboard. Visualizing the error makes tuning significantly easier.
- **Iterative Changes:** Change one variable at a time (e.g., only adjust Translational P).
- **Start Small:** Start with very small values, especially for `D` and `I`.
- **Battery Voltage:** A fresh battery vs. a dying battery will change how the robot responds. Try to tune with a battery at a realistic match voltage (e.g., around 13.0V - 13.5V).
- **Verify with the Visualizer:** If your physical robot struggles to follow a path smoothly even after tuning, use the Pedro Pathing Visualizer to ensure the path itself doesn't exceed the robot's physical constraints (Max Velocity, Max Acceleration).
